# Architecture Decision Records

ADRs capture important technical choices and their trade-offs.

Each ADR should use this structure:

```text
# NNNN - Title

## Status

Proposed | Accepted | Superseded

## Context

What problem or constraint forced a decision?

## Decision

What did we choose?

## Alternatives

What else did we consider?

## Consequences

What benefits, risks, and future costs does this decision create?
```
